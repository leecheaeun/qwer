# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정(Circle, AABB, OBB-SAT) 구현
- 회전하는 오브젝트 + 키 입력(Z)으로 속도 변화
- 각 충돌 결과를 화면에 동시에 표시하여 비교

## 관찰 내용

### 원형 vs AABB
원형 충돌은 중심 거리만으로 계산하기 때문에 가장 단순하고 빠르지만 실제 사각형 모양과 맞지 않는 경우가 많았다 특히 사각형의 모서리 부분에서는 실제로는 닿지 않았는데도 충돌로 판정되는 경우가 있었고 반대로 살짝 닿았는데도 놓치는 경우는 거의 없었다 AABB는 축에 정렬된 사각형 기준이라 직관적이고 원형보다 정확했지만 오브젝트가 회전할 경우 실제보다 더 큰 영역을 차지하게 되어 충돌이 과하게 발생하는 문제가 있었다

### AABB vs OBB
AABB는 회전된 오브젝트를 감싸는 방식이라 회전 각도가 커질수록 박스가 불필요하게 커져서 충돌 판정이 부정확해지는 것을 확인했다 반면 OBB는 오브젝트의 회전을 그대로 반영하기 때문에 실제 모양과 거의 일치하는 정확한 충돌 판정이 가능했다 다만 SAT 알고리즘을 사용하면서 계산량이 증가하기 때문에 성능 비용이 더 크다는 점도 함께 알 수 있었다

## AI와의 대화에서 배운 것
단순한 충돌 판정이라도 방식에 따라 정확도와 성능이 크게 달라진다는 점이 인상적이었고 특히 SAT 알고리즘이 모든 축에 대해 투영해서 겹침을 검사한다는 개념이 흥미로웠다 또한 AABB는 빠르지만 부정확하고 OBB는 정확하지만 비용이 크기 때문에 상황에 따라 적절한 방식을 선택해야 한다는 점이 중요하다고 느꼈다

## 내 게임에 적용한다면
내가 게임을 만든다면 기본적인 충돌은 AABB를 사용하고 캐릭터나 중요한 오브젝트끼리의 충돌에는 OBB를 사용할 것 같다 이유는 전체적인 성능을 유지하면서도 중요한 상호작용에서는 정확한 판정이 필요하기 때문이다 그리고 단순한 거리 기반 판정이 필요한 경우 예를 들어 범위 체크나 근접 감지에는 원형 충돌을 함께 사용할 것이다
